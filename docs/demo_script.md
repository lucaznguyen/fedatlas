# FedAtlas Demo Script

## 3-5 Minute Presentation

Hello, today I am presenting FedAtlas: Mapping the Global Research-to-Code Ecosystem of Federated Learning.

Start on the Overview tab. The KPI cards summarize the size of the field: papers, authors, institutions, countries, venues, GitHub repositories, citations, stars, and the overall research-to-code score. The time series shows how Federated Learning has grown since 2016, and the GitHub-linked line shows whether implementation availability is keeping pace with publication volume.

Move to the topic heatmap. This view shows that Federated Learning is not one uniform area. Core algorithms, privacy, robustness, edge systems, healthcare, wireless networking, and newer foundation-model topics evolve at different speeds.

Next, open Global Collaboration. The map and collaboration tables show where research activity is concentrated and which countries bridge communities. This is important because high publication volume and high bridge centrality are not always the same thing.

Open Network Explorer. This graph connects papers, authors, institutions, countries, topics, venues, repositories, and contributors. Filtering the graph prevents overload and helps focus on the most central actors or topics.

Now switch to Research-to-Code Gap. The scatterplot compares citation impact with code impact. Highly cited papers without GitHub links represent a reproducibility or transfer gap. High code-impact papers show where implementations are shaping practice even when citation patterns differ.

Finish with the key insight: Federated Learning is growing quickly, but collaboration structure and code adoption are uneven across topics, countries, and venues. FedAtlas makes those uneven patterns visible and updateable for future weekly crawls.
